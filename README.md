# Socket Programming – Distributed Job Queue (Jackfruit Mini Project)

A secure, multi-client distributed job queue implemented with **raw TCP sockets** and **SSL/TLS**, in Python.

---

## Architecture
<img width="1024" height="477" alt="image" src="https://github.com/user-attachments/assets/0a4d1360-7797-4f34-bef2-c8df92370c32" />

```
  ┌──────────┐         TLS/TCP          ┌─────────────────────────────────┐
  │  Client  │ ──── JSON job ────────►  │                                 │
  │ (client) │ ◄─── JOB_RECEIVED ─────  │           SERVER                │
  └──────────┘                          │  • Accepts clients & workers    │
                                        │  • Maintains job queue          │
  ┌──────────┐         TLS/TCP          │  • Tracks in-progress jobs      │
  │  Worker  │ ──── WORKER ──────────►  │  • Re-queues on worker crash    │
  │ (worker) │ ◄─── READY ───────────   │  • Reports stats every 10s      │
  │          │ ──── GET_JOB ─────────►  │                                 │
  │          │ ◄─── JSON job ─────────  └─────────────────────────────────┘
  │          │ ──── DONE/FAILED ─────►
  └──────────┘
```

### Communication Protocol

| Step | Direction | Message |
|------|-----------|---------|
| Client submits job | Client → Server | `{"job_id": N, "task": "..."}` |
| Server ACKs | Server → Client | `"JOB_RECEIVED"` |
| Worker identifies | Worker → Server | `"WORKER"` |
| Server signals | Server → Worker | `"READY"` |
| Worker requests job | Worker → Server | `"GET_JOB"` |
| Server dispatches | Server → Worker | JSON job or `"NO_JOB"` |
| Worker reports | Worker → Server | `"DONE"` or `"FAILED"` |

All messages are exchanged over **TLS 1.2+ encrypted TCP sockets**.

## Files

| File | Purpose |
|------|---------|
| `server.py` | Central server — job queue, worker dispatch, stats |
| `client.py` | Submits jobs to the server |
| `worker.py` | Polls server for jobs, processes them, reports result |
| `benchmark.py` | Throughput and latency performance evaluation |
| `gen_certs.sh` | Generates self-signed SSL certificate for local dev |

## Setup

### 1. Prerequisites

- Python 3.8+
- OpenSSL (for certificate generation)

### 2. Generate SSL Certificate

```bash
bash gen_certs.sh
```

This creates `cert.pem` (certificate) and `key.pem` (private key) in the current directory.

### 3. Start the Server

```bash
python server.py
```

### 4. Start One or More Workers (in separate terminals)

```bash
python worker.py
python worker.py   # run as many workers as needed
```

### 5. Send Jobs from the Client

```bash
python client.py
```

### 6. Run the Performance Benchmark

```bash
python benchmark.py
```

---

## Key Features

- **SSL/TLS encryption** on all socket connections (TLS 1.2 minimum)
- **Concurrent clients and workers** via Python threading
- **Job re-queuing** — if a worker crashes or times out, its job is automatically re-enqueued
- **Worker crash simulation** — 20% random crash rate to demonstrate fault tolerance
- **Performance metrics** — built-in benchmark measuring throughput, mean/median/p95 latency
- **Structured logging** with timestamps across all components
- **Graceful error handling** — SSL errors, timeouts, malformed JSON, abrupt disconnections

---

## Security Notes

- The self-signed certificate generated by `gen_certs.sh` is suitable for local development and demos only.
- For production use, replace `cert.pem` / `key.pem` with certificates from a trusted CA.
- `check_hostname = False` is set because we connect via IP (`127.0.0.1`); set to `True` with a proper domain in production.
